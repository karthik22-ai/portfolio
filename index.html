Project Overview

Aidex is an AI-powered medical assistant designed to provide empathetic, multilingual, and medically safe interactions. It uses Google Gemini APIs for natural language and vision-based understanding, with a multi-agent architecture for handling specific roles like safety filtering, symptom analysis, and translation.

Core Features

  • Medical Safety Guard Agent: Filters incoming queries and blocks non-medical or unsafe requests. Ensures the system only engages in health-related conversations.
  • Symptom Analysis Agent: Engages in empathetic, chatty, and patient-friendly conversations. Collects symptoms, suggests possible causes, and provides structured responses.
  • Language Translation Agent: Supports multilingual conversations by translating user queries and assistant responses. Expands accessibility across global users.
  • Vision-based Analysis: Uses Gemini Vision models to process webcam input or uploaded images. Can assist in detecting abnormalities in scans/reports (future-ready).
  • Real-time Communication: Backend built with FastAPI + WebSockets for low-latency interactions. Supports streaming responses for smooth conversational flow.

System Workflow

The system follows a structured workflow to ensure safe and effective medical assistance:

  1. User sends a medical query (via text, speech, or image).
  2. Medical Guard Agent checks if it's a valid medical question.
  3. If valid â†’ request goes to Symptom Analysis Agent, which responds empathetically.
  4. If user speaks another language â†’ Translation Agent bridges communication.
  5. If an image is provided â†’ Vision Analysis (Gemini Vision) extracts medical context.
  6. The backend streams the structured response back to the user.

Brain Tumor Detection Using MRI Images

Deep learning solution for accurate brain tumor detection from MRI scans

Project Overview

This project focuses on developing a deep learning model to detect brain tumors from MRI images. The solution addresses the critical need for accurate and early tumor detection, which can significantly improve patient outcomes. By leveraging convolutional neural networks (CNNs), the model can analyze MRI scans and identify tumor regions with high precision.

Key Features

  • Custom Dataset Creation: Compiled a comprehensive dataset from multiple hospitals, ensuring diverse and representative samples.
  • Advanced Preprocessing: Implemented noise removal, image enhancement, and resolution improvement techniques to optimize input data.
  • High Accuracy Model: Achieved 96% accuracy in tumor detection using VGG16 architecture with transfer learning.
  • Visualization Tools: Created heatmap visualizations to highlight tumor regions for medical professionals.
  • Performance Optimization: Optimized model for faster inference while maintaining high accuracy.

Technical Approach

The project follows a systematic approach to tumor detection:

  1. Data collection from multiple hospital sources with proper labeling.
  2. Image preprocessing including noise reduction, contrast enhancement, and normalization.
  3. Model architecture selection based on VGG16 with transfer learning.
  4. Training with data augmentation to improve model generalization.
  5. Implementation of Grad-CAM for visualization of tumor regions.
  6. Performance evaluation using accuracy, precision, recall, and F1-score metrics.

Fine-Tuning Phi-3 Mini with Unsloth

Efficient fine-tuning of Microsoft's Phi-3 Mini LLM with low VRAM optimization

Project Overview

This project focuses on fine-tuning Microsoft's Phi-3 Mini language model using Unsloth, a library designed for efficient training with reduced memory requirements. The goal was to adapt the model for specific tasks while optimizing for low VRAM usage, making it accessible for developers with limited hardware resources. The project demonstrates advanced techniques in parameter-efficient fine-tuning and quantization.

Key Features

  • Low VRAM Optimization: Implemented techniques to reduce memory usage during training, enabling fine-tuning on consumer-grade hardware.
  • PEFT Implementation: Utilized Parameter-Efficient Fine-Tuning (PEFT) methods to train only a small subset of parameters.
  • Quantization Techniques: Applied 4-bit and 8-bit quantization to further reduce memory requirements without significant performance loss.
  • Custom Dataset Preparation: Created and processed datasets tailored for specific fine-tuning tasks.
  • Performance Evaluation: Conducted comprehensive evaluations to measure model performance before and after fine-tuning.
  • Documentation & Reproducibility: Provided detailed Jupyter notebooks with step-by-step instructions for reproducibility.

Technical Approach

The project employed several advanced techniques to achieve efficient fine-tuning:

  1. Setup of the Unsloth environment with optimized dependencies.
  2. Loading the Phi-3 Mini model with 4-bit quantization to minimize memory footprint.
  3. Implementation of LoRA (Low-Rank Adaptation) for parameter-efficient fine-tuning.
  4. Preparation of custom datasets with appropriate formatting for the model.
  5. Configuration of training parameters for optimal performance with limited VRAM.
  6. Execution of the fine-tuning process with memory-efficient techniques.
  7. Evaluation of the fine-tuned model using relevant metrics and benchmarks.
